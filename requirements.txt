================================================================================
1. PROJECT OVERVIEW
================================================================================
The Event Management System allows customers to browse event services, view
packages, submit inquiries, and book events online.

Supported event categories:
  - Wedding Events
  - Birthday Parties
  - Corporate Events

Administrators can manage bookings, event packages, customer inquiries,
galleries, and website content through a secure admin panel.

7. DEVELOPMENT PLAN (ESTIMATED: 2–3 WEEKS)
================================================================================
Phase 1 — Setup & Foundation (Days 1–3)
  - Install Laravel, configure .env and MySQL connection
  - Set up Git repository and project structure
  - Create database migrations: users, categories, packages,
    bookings, inquiries, gallery, testimonials
  - Define Eloquent models and relationships
  - Seed initial categories (Wedding, Birthday, Corporate)

Phase 2 — Authentication & Admin Foundation (Days 4–5)
  - Implement Laravel authentication (login/logout)
  - Add role-based access (admin vs customer)
  - Build admin dashboard layout with statistics

Phase 3 — Admin CRUD Modules (Days 6–10)
  - Categories management (CRUD)
  - Packages management (CRUD + image upload)
  - Bookings management (view, status update, delete)
  - Inquiries management (view, reply, delete)
  - Gallery management (image/video upload, edit, delete)
  - Testimonials management (CRUD)
  - User management (manage users, change password)

Phase 4 — Frontend Public Pages (Days 11–15)
  - Home, About Us, Services pages
  - Packages page (dynamic from DB) with Book Now
  - Gallery page with category filter
  - Testimonials page
  - Contact page with form validation
  - Book Event page (booking form -> bookings table)

Phase 5 — Features, Polish & Testing (Days 16–18)
  - Responsive / mobile-friendly styling
  - SEO meta tags and friendly URLs
  - Search & filter functionality
  - Form validation (client + server side)
  - Cross-browser and device testing
  - Bug fixes and final review

Phase 6 — Deployment (Days 19–21)
  - Production environment configuration
  - Database migration to production
  - Final QA and handover
